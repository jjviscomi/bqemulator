# Code of Conduct

This project adopts the
[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)
as its code of conduct.

The full text is available at the link above and applies to all project spaces
(GitHub issues, pull requests, discussions, the documentation site, and any
other channels maintained by this project).

## Reporting

To report a Code of Conduct violation, **open a private report through
the repository's Security tab**:

1. Navigate to the
   [project repository on GitHub](https://github.com/jjviscomi/bqemulator).
2. Click the **Security** tab.
3. Choose **Report a vulnerability** and tick the
   *"This isn't a security issue — it's about behavior"* note in the
   report body so triage routes the report to the conduct workflow
   rather than the security workflow.

GitHub's private reporting flow notifies the project maintainers
directly while keeping the report invisible to the rest of the
community until the maintainer chooses to act on it. The maintainer
will follow up via the private advisory thread (not email).

If the report involves a maintainer, the report stays private to the
remaining maintainer set; private advisories are scoped per-recipient.

This channel intentionally avoids putting a personal email address in
the public repository (and avoids spam farming of any project-owned
alias). The same channel is used for software-security vulnerability
reports — see [`SECURITY.md`](SECURITY.md) — and the maintainer
distinguishes the two using the note in the report body.

All reports are received by the project maintainers and handled
confidentially.
